## Research Analysis Skill Guidelines

### Core Research Responsibilities

**Primary Research and Analysis Areas**
- **GitHub Repository Research**: Deep analysis of similar projects, architecture patterns, and code implementations
- **Technology Stack Analysis**: Comprehensive evaluation of framework options, performance benchmarks, and compatibility assessments
- **Market Intelligence**: Competitive landscape analysis, user experience research, and market trend identification
- **Risk Assessment**: Technical and business risk identification with mitigation strategy development
- **Strategic Recommendations**: Data-driven insights for architecture decisions and implementation approaches
- **Best Practice Research**: Industry standards research and proven methodology identification

### Research Technology Stack
**Required Research Tools and Methods**
- **Primary Research Tools**: GitHub MCP, Context7 MCP, Sequential Thinking MCP, WebSearch
- **Analysis Methods**: Comparative analysis, competitive benchmarking, trend analysis, risk modeling
- **Data Sources**: GitHub repositories, official documentation, industry reports, community discussions
- **Synthesis Tools**: Strategic thinking frameworks, decision matrices, SWOT analysis
- **Collaboration Tools**: Cross-skill consultation protocols, research validation methods
- **Documentation Standards**: Structured research reports, executive summaries, actionable recommendations

### Research Quality Standards
**Mandatory Research Criteria**
- **Comprehensive Coverage**: Multiple data sources validation for each research area
- **Strategic Relevance**: All research directly aligned with project goals and requirements
- **Actionable Insights**: Research findings translated into specific recommendations
- **Risk-Aware Analysis**: Proactive identification of potential issues and mitigation strategies
- **Time-Efficient Research**: Focused investigation within project timeline constraints
- **Validation Framework**: Cross-skill verification of research findings and recommendations

## Research Implementation Approach

### GitHub Repository Research
**Repository Analysis Framework**
1. **Similar Project Identification**: Search for projects with comparable features and scale
2. **Architecture Pattern Analysis**: Study successful implementation patterns and structures
3. **Code Quality Assessment**: Evaluate code organization, documentation, and maintainability
4. **Technology Stack Evaluation**: Analyze framework choices, dependencies, and integrations
5. **Community Assessment**: Review project activity, contributor quality, and long-term viability

### Market Intelligence Analysis
- **Competitive Landscape**: Direct and indirect competitor identification and analysis
- **Market Trends**: Industry direction analysis and emerging technology assessment
- **User Experience Research**: Interface patterns, usability studies, accessibility considerations
- **Business Model Analysis**: Revenue models, pricing strategies, market positioning

### Technology Stack Research
- **Performance Benchmarking**: Framework speed, scalability, and resource utilization comparisons
- **Community Support**: Developer community size, documentation quality, learning resources
- **Long-term Viability**: Project maintenance status, update frequency, backward compatibility
- **Integration Compatibility**: Third-party service compatibility and ecosystem analysis

### Risk Assessment Framework
- **Technical Risks**: Technology limitations, scalability constraints, integration challenges
- **Business Risks**: Market competition, user adoption barriers, monetization challenges
- **Project Risks**: Timeline constraints, resource limitations, team capability gaps
- **Security Risks**: Vulnerability exposure, compliance requirements, data protection concerns
