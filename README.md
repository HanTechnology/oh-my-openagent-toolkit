## FAQ

**Q: Do I need to configure anything manually?**
A: No! Just install the skills in Claude Code and describe your project. Everything else is automatic.

**Q: Can I use only some skills, not all 13?**
A: Yes! Skills are contextually invoked. Only relevant skills activate for your project type.

**Q: What if I want to use a different backend framework (not NestJS/FastAPI)?**
A: You can create a new backend skill following the template. Contributions welcome!

**Q: How do skills know when to activate?**
A: Claude analyzes context and automatically invokes skills based on description matching. No manual activation needed.

**Q: Can I override technology choices?**
A: Yes! Mention specific preferences in your request: "Create a Next.js app using Prisma instead of TypeORM."

**Q: What happens if a skill makes a mistake?**
A: Quality-controller validates all outputs. If errors are detected, the responsible skill is re-invoked to fix issues.

**Q: Is this only for new projects?**
A: No! Skills can analyze and enhance existing projects. Just describe what you want to improve.

**Q: How long does it take to build a complete app?**
A: Simple apps: 10-15 minutes. Medium complexity: 20-30 minutes. Complex apps: 30-60 minutes. All production-ready.

**Q: Do I need Claude Pro for this?**
A: Claude Code with any plan works. Sonnet 4.5 model recommended for best results.

**Q: Can I use this commercially?**
A: Yes! MIT License allows commercial use. See LICENSE for details.
